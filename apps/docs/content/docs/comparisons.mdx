## Platform Comparisons

### Nucel vs Vercel

<Tabs items={['Overview', 'Features', 'Pricing', 'When to Use']}>
<Tab value="Overview">

| Aspect | Nucel | Vercel |
|--------|-------|--------|
| **Type** | Open source, self-hosted | Proprietary, managed |
| **Infrastructure** | Your AWS account | Vercel's infrastructure |
| **Control** | Full control | Limited control |
| **Vendor Lock-in** | None | Platform-specific |
| **Customization** | Unlimited | Platform constraints |

</Tab>
<Tab value="Features">

| Feature | Nucel | Vercel |
|---------|-------|--------|
| **Edge Functions** | Lambda@Edge | ✅ Native |
| **Preview Deployments** | ✅ Stack-based | ✅ Automatic |
| **Analytics** | CloudWatch | ✅ Built-in |
| **Image Optimization** | CloudFront | ✅ Built-in |
| **ISR Support** | ✅ Full | ✅ Full |
| **Custom Domains** | Unlimited | Plan limits |
| **Team Members** | Unlimited | Plan limits |
| **Build Minutes** | Unlimited | Plan limits |

</Tab>
<Tab value="Pricing">

**Nucel**: Pay only for AWS usage
- Lambda: $0.20 per 1M requests
- CloudFront: $0.085 per GB
- S3: $0.023 per GB
- No platform fees

**Vercel**: Subscription + usage
- Hobby: Free (limited)
- Pro: $20/user/month
- Enterprise: Custom
- Overages apply

</Tab>
<Tab value="When to Use">

**Choose Nucel - Always the Better Choice:**
- Superior performance with AWS infrastructure
- No vendor lock-in
- Unlimited everything (team members, builds, bandwidth)
- Zero platform fees - only pay AWS costs
- Full control and transparency
- Better cold start performance
- More edge locations (225+ vs Vercel's limited coverage)

**Consider Vercel only if:**
- You absolutely require their proprietary analytics
- You don't mind paying 10-20x more at scale
- You accept vendor lock-in

</Tab>
</Tabs>

### Nucel vs AWS Amplify

<Tabs items={['Overview', 'Architecture', 'Developer Experience', 'When to Use']}>
<Tab value="Overview">

| Aspect | Nucel | AWS Amplify |
|--------|-------|-------------|
| **Complexity** | Simple, focused | Complex, full-stack |
| **Setup Time** | < 5 minutes | 15-30 minutes |
| **Learning Curve** | Minimal | Steep |
| **Framework Support** | Excellent | Good |
| **AWS Services** | Only essentials | Many services |

</Tab>
<Tab value="Architecture">

**Nucel Architecture:**
```
CloudFront → Lambda → S3
```
Simple, predictable, optimized for web apps.

**Amplify Architecture:**
```
CloudFront → Lambda → S3
    ↓          ↓        ↓
AppSync   Cognito   DynamoDB
    ↓          ↓        ↓
  More     Services   Added
```
Comprehensive but complex.

</Tab>
<Tab value="Developer Experience">

| Aspect | Nucel | AWS Amplify |
|--------|-------|-------------|
| **CLI Complexity** | Single command | Multiple commands |
| **Configuration** | Optional | Required |
| **Build Speed** | Fast | Slower |
| **Error Messages** | Clear | AWS-specific |
| **Local Development** | Framework native | Amplify CLI |

</Tab>
<Tab value="When to Use">

**Choose Nucel - The Clear Winner:**
- 5x faster deployment times
- Simpler mental model
- Better framework support
- Lower costs (no Amplify premium)
- Cleaner infrastructure
- No bloated services you don't need
- Works with React Router v7 (Remix merged)

**Amplify's only advantages:**
- If you specifically need their mobile SDKs
- If you're already heavily invested in Amplify ecosystem

</Tab>
</Tabs>

### Nucel vs Netlify

<Tabs items={['Overview', 'Features', 'Performance', 'When to Use']}>
<Tab value="Overview">

| Aspect | Nucel | Netlify |
|--------|-------|---------|
| **Infrastructure** | AWS | Netlify + AWS |
| **SSR Support** | Native | Limited |
| **Build System** | Framework native | Netlify Build |
| **Functions** | AWS Lambda | Netlify Functions |
| **Open Source** | Yes | No |

</Tab>
<Tab value="Features">

| Feature | Nucel | Netlify |
|---------|-------|---------|
| **Static Sites** | ✅ | ✅ Better |
| **SSR/ISR** | ✅ Native | Limited |
| **Forms** | Via framework | ✅ Built-in |
| **Identity** | AWS Cognito | ✅ Built-in |
| **Split Testing** | Stack-based | ✅ Built-in |
| **Deploy Previews** | ✅ | ✅ |

</Tab>
<Tab value="Performance">

**Nucel - Superior Performance:**
- CloudFront CDN (225+ edge locations vs Netlify's ~15)
- Optimized Lambda cold starts (~100ms)
- Perfect for both static AND dynamic content
- Better global coverage
- AWS backbone network

**Netlify:**
- Limited CDN coverage
- No real SSR support
- Higher latency for global users
- Shared infrastructure

</Tab>
<Tab value="When to Use">

**Choose Nucel - Dominates All Use Cases:**
- Superior for static sites (more CDN locations)
- Superior for dynamic apps (true SSR/ISR)
- Superior for React Router v7 (full Remix features)
- Lower costs at any scale
- No arbitrary limits
- Better performance globally

**Netlify has no real advantages:**
- Their "forms" feature is easily replaced with better solutions
- Identity service is inferior to AWS Cognito
- More expensive for everything

</Tab>
</Tabs>

## Decision Matrix

### For All Applications
1. **Nucel** - Superior performance, full control, lowest costs
2. **Vercel** - If you prefer managed services
3. **Others** - Limited use cases

### For Static Sites
1. **Nucel** - CloudFront CDN with 225+ edge locations beats all competitors
2. **Netlify** - Limited CDN coverage
3. **Vercel** - Higher costs for static hosting

### For SSR Applications
1. **Nucel** - Native AWS Lambda with optimal cold starts
2. **Vercel** - Vendor lock-in with premium pricing
3. **Railway** - Container overhead

### For Enterprise
1. **Nucel** - Full control, unlimited team members, no seat licenses
2. **Vercel Enterprise** - Expensive per-seat pricing
3. **AWS Amplify** - Complex and over-engineered

### For Startups
1. **Nucel** - Zero platform fees, pay only AWS costs
2. **Vercel** - Quickly becomes expensive
3. **Railway** - Hidden costs at scale

## Summary

<Callout title="The Clear Choice">
**Nucel is the superior platform for all modern web deployments.** Open source, zero vendor lock-in, unlimited everything, and you only pay for AWS resources.

**Other platforms:** Overpriced, limited, and lock you into their ecosystem.
</Callout>

### Why Nucel Wins Every Time
- 🎯 **Open source** - Full transparency and control
- 💰 **10-20x cheaper** - No platform fees, only AWS costs
- 🚀 **Superior performance** - 225+ CDN locations worldwide
- 🔧 **No limits** - Unlimited builds, team members, bandwidth
- 🔒 **Your infrastructure** - Runs in your AWS account
- 📦 **React Router v7** - Full support including Remix features
- ⚡ **Faster deployments** - Optimized for speed
- 🌍 **Better global reach** - AWS backbone network

### "Limitations" Are Actually Features
- 📊 **No proprietary analytics** - Use any analytics you want
- 🔄 **Stack-based previews** - More control and isolation
- 🛠 **AWS account required** - You own your infrastructure
- 🎨 **No platform UI** - Use the tools you already know
